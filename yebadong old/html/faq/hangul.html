E-Mail에서의 한글 문제

Hangul Troubles in E-Mail

  • 지금 가장 문제가 되고 있는것이 메일을 주고 받을때 한글이 보이지 않는 문제 인것 같습니다. 몇몇 분들께서 왕창 깨진 한글 파일을 받으셨을줄 압니다.

    For the present, the most serious problem is the HANGUL problem. I think some of you got broken korean mails.

  • 제가 몇몇 도사님들께 문의해본 결과 다음과 같이 그 문제를 해결할 수 있을 듯 하니 참고하시기 바랍니다.

    I asked to some wizards to fix hangul problem and I think you can fix that problem. Please consider below.

    1. 한글 메일(hmail)을 사용하시는 겁니다. 이 방법은 적당한 ftp site 에 가셔서 구하셔야 하는데 좀 귀찮을 듯 하군요. 하지만 가장 확실하고, 좋은 방법입니다. 현재 구할 수 있는 가장 높은 sendmail version은 sendmail.8.6.12h2.tar.gz 입니다.

    First method is to use hmail(hangul mail). I think this method is time consuming and may bother you. But, best and most precise way. Get sendmail.8.6.12h2.tar.gz from (Korean) ftp site, and install it. Most famous ftp site is "cair-archive.kaist.ac.kr:./hangul/mail/"

    2. hcode 와 같은 code converter 용 utility 를 사용하는 겁니다. 이 방법은 역시 ftp site 에 가셔서 hcode 나 ks2ioc ios2ks 와 같은 유틸리티를 구하셔서 /usr/local/bin 같은 디렉토리에 깔아 주시고 hcode 인 경우, ~/.mailrc 파일에 다음의 내용을 추가해 주시면 됩니다.

      set PAGER = "/usr/local/bin/hcode -ik | /usr/local/bin/more"
    이 명령은 mail 명령으로 file 을 볼때 한글이 안깨지도록 하기 위한 것입니다.

    Second method is to use file converter such as "hcode". First get "hcode" or "ks2ios/ios2ks" utilities in ftp site, then install them to /usr/local/bin. In case of "hcode", you should subscribe the following contents on ~/.mailrc file.

      set PAGER = "/usr/local/bin/hcode -ik | /usr/local/bin/more "
    This command is for unbroken hangul file when you see the mailed file which is sent in ISO code.

    전송 받으신 화일을 저장할 때에도 저장된 파일은 여전히 ISO 코드 이므로, 그 파일을 hcode 등을 이용해서 KS 코드로 바꾸어 주면 됩니다.

    If you save the mailed file, the saved file is stiil in ISO code. So, you must convert that file to KS code by using "hcode" or so.

    3. elm 을 사용하시는 분들은 옵션 'o' 를 선택하시고 다음과 같이 세팅해 주시면 됩니다.

    If you are using "elm", select 'o' in menu, then set as follows.

      D)isplay mail using : /usr/loca/bin/hcode -ik | /usr/local/bin/more

      ....

      P)rint mail using : /usr/local/bin/hcode -ik >> save.file

    위의 세팅은 print 명령을 이용하여 파일의 내용을 save.file 이란 파일에 차례로 붙이게 되므로 2 의 경우와 같은 conversion 이 필요가 없게 됩니다.

    If you set as above, elm saves the mailed file in "save.file" in sequence, you don't need to convert the file to KS code.
